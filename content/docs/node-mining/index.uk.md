Цей посібник описує, як отримувати винагороди TALER через стейкінг Proof-of-Stake (PoS) з використанням GUI гаманця.

## Розуміння консенсусу TALER

TALER використовує гібридний механізм консенсусу Proof-of-Work (PoW) і Proof-of-Stake (PoS):

- **Proof-of-Work (PoW)**: Майнери генерують блоки з використанням обчислювальної потужності
- **Proof-of-Stake (PoS)**: Власники монет стейкають свої монети для генерації блоків

**Цей посібник зосереджений на PoS стейкінгу**, який є рекомендованим методом для більшості користувачів, оскільки він:
- Не вимагає спеціалізованого обладнання
- Споживає мінімум електроенергії
- Дозволяє отримувати пасивні винагороди на ваші існуючі холдинги TALER
- Допомагає захищати мережу

---

## Що таке PoS стейкінг?

### Як працює стейкінг

Proof-of-Stake дозволяє вам заробляти винагороди, тримаючи монети TALER у вашому гаманці. Замість використання обчислювальної потужності, як у PoW майнінгу, PoS використовує:

- **Вік монет**: Як довго ви тримаєте монети
- **Кількість монет**: Скільки монет ви тримаєте
- **Ймовірність**: Старші та більші суми мають вищу ймовірність стейкінгу

Коли ваші монети успішно стейкаються, ви створюєте новий блок і отримуєте:
- **Винагорода за блок**: Базова PoS винагорода (30% від поточної субсидії PoW блоку)
- **Комісії за транзакції**: Всі комісії від транзакцій у вашому блоці

### Вимоги для стейкінгу

Для стейкінгу монет TALER вам необхідно:

1. **Мінімальна сума**: Мінімум 10 TLR на один вхід монет
2. **Мінімальний вік**: Монети повинні дозріти протягом 48 годин після отримання
3. **Зашифрований гаманець**: Гаманець повинен бути зашифрований для безпеки
4. **Розблокований гаманець**: Гаманець повинен бути розблокований для стейкінгу
5. **Працюючий гаманець**: taler-qt повинен бути запущений і синхронізований

**Важливо**: Монети з менш ніж 10 TLR не можуть стейкатися. Монети стають доступними для стейкінгу через 48 годин після їх отримання та підтвердження.

## Налаштування PoS стейкінгу

### Крок 1: Зашифрувати ваш гаманець

**Чому спочатку шифрувати?**
- Стейкінг вимагає зашифрованого гаманця
- Захищає ваші кошти від несанкціонованого доступу
- Кращі практики безпеки перед стейкінгом

**Як зашифрувати**:

1. **Відкрийте гаманець taler-qt**
2. **Перейдіть до меню Settings**
3. **Натисніть "Encrypt Wallet"**
4. **Введіть сильну парольну фразу**
   - Рекомендується мінімум 20+ символів
   - Суміш літер, цифр, символів
   - Запишіть її надійно - **ви не можете відновити забуті парольні фрази**
5. **Підтвердіть парольну фразу**
6. **Дочекайтеся завершення шифрування**

Гаманець перезапуститься після шифрування.

**Вже зашифрований?** Переходьте до кроку 2.

### Крок 2: Поповнити ваш гаманець

Для стейкінгу вам потрібні монети TALER у вашому гаманці:

1. **Згенеруйте адресу отримання** у вкладці Receive
2. **Переведіть TALER** у ваш гаманець з біржі або іншого гаманця
3. **Дочекайтеся підтверджень** (рекомендується 6+)
4. **Почекайте 48 годин**, поки монети дозріють

Ваші монети повинні дозрівати протягом 48 годин, перш ніж вони стануть доступними для стейкінгу.

### Крок 3: Відкрити консоль відладки

Консоль відладки дозволяє вам виконувати команди для управління стейкінгом.

**Для доступу до консолі відладки**:

1. **Відкрийте гаманець taler-qt**
2. **Перейдіть до меню Help**
3. **Натисніть "Debug Window"**
4. **Виберіть вкладку "Console"**

Ви побачите інтерфейс командного рядка, де можна вводити RPC команди.

### Крок 4: Розблокувати гаманець для стейкінгу

Щоб увімкнути стейкінг, ви повинні розблокувати ваш зашифрований гаманець за допомогою консолі.

**Команда**:
```
walletpassphrase "your-passphrase" timeout
```

**Параметри**:
- `"your-passphrase"`: Ваша парольна фраза шифрування гаманця (в лапках)
- `timeout`: Кількість секунд для підтримання гаманця розблокованим

**Приклади**:

```
walletpassphrase "MySecurePassphrase123!" 2592000
```
Це розблокує гаманець на 2,592,000 секунд (30 днів).

```
walletpassphrase "MySecurePassphrase123!" 31536000
```
Це розблокує гаманець на 31,536,000 секунд (1 рік).

**Керівництво по таймауту**:
- **Короткостроковий**: 86400 (1 день)
- **Середньостроковий**: 2592000 (30 днів)
- **Довгостроковий**: 31536000 (1 рік)
- **Максимум**: 100000000 (~3.17 років)

**Після розблокування**:
- Стейкінг починається автоматично
- Гаманець автоматично заблокується після закінчення таймаута
- Вам потрібно буде розблокувати знову для продовження стейкінгу після таймаута

## Моніторинг вашого стейкінгу

### Перевірка статусу гаманця

Щоб перевірити, що ваш гаманець розблокований і готовий до стейкінгу:

**Команда**:
```
getwalletinfo
```

**Важливі поля**:
- `"unlocked_until"`: Часова мітка, коли гаманець заблокується знову (0 = заблокований)
- `"balance"`: Ваш доступний баланс
- `"immature_balance"`: Очікуючі винагороди за стейкінг (потрібні підтвердження)

**Приклад виводу**:
```json
{
  "walletname": "",
  "walletversion": 169900,
  "balance": 1000.00000000,
  "immature_balance": 2.10000000,
  "unlocked_until": 1748362000,
  ...
}
```

Якщо `unlocked_until` дорівнює 0, ваш гаманець заблокований - розблокуйте його знову за допомогою `walletpassphrase`.

### Перевірка доступних монет для стейкінгу

Щоб побачити, які монети доступні для стейкінгу:

**Команда**:
```
listunspent
```

Це показує всі невитрачені монети у вашому гаманці з:
- `amount`: Значення монети
- `confirmations`: Кількість підтверджень
- `spendable`: Чи може монета бути витрачена/застейкана

**Придатні для стейкінгу**:
- Amount >= 10 TLR
- Confirmations >= 288 блоків (~48 годин при 60-секундних блоках для PoW)
- Spendable = true

### Розуміння віку монет

**Вік монет визначає вагу стейкінгу**:

- **Мінімальний вік**: 48 годин (монети стають доступними)
- **Максимальний вік**: 90 днів (монети досягають максимальної ваги)
- **Тік віку**: 1 день (вага збільшується щодня)

**Як це працює**:
- День 0-2: Не придатні для стейкінгу
- День 2-90: Вага стейкінгу збільшується щодня
- День 90+: Досягнута максимальна вага (подальшого збільшення немає)

Чим довше ви тримаєте монети без витрат, тим вища ваша ймовірність стейкінгу.

## Винагороди за стейкінг

### Розрахунок винагороди

Винагороди за PoS стейкінг засновані на:

1. **Вік монет**: Як довго ви тримали монети (у монето-днях)
2. **Кількість монет**: Скільки монет ви стейкаєте
3. **Субсидія блоку**: Поточна PoS винагорода за блок (30% від PoW субсидії)
4. **Мережевий запас**: Загальний запас TALER

**Формула**:
```
Винагорода = (Вік монет × Річна ставка монет) + Комісії за транзакції
```

Винагорода розраховується автоматично, коли ви успішно стейкаєте блок.

### Очікувана дохідність

**Типові винагороди за стейкінг**:
- Базова PoS винагорода за блок: 2.1 TLR (30% від 7 TLR PoW блоку)
- Плюс комісії за транзакції з блоку
- Скориговані з урахуванням віку монет і мережевих умов

**Частота стейкінгу**:
- Залежить від кількості та віку ваших монет
- Більші холдинги стейкаються частіше
- Складність мережі впливає на всіх стейкерів однаково

**Приклад**:
- 1,000 TLR застейкано
- 90 днів віку (максимальна вага)
- Може стейкатися кожні кілька днів або тижнів
- Кожен стейк приносить ~2.1 TLR базової винагороди

### Коли з'являються винагороди

**Після стейкінгу блоку**:

1. **Негайно**: Винагорода з'являється в "Immature Balance"
2. **Дозрівання**: Потребує 288 підтверджень (~48 годин)
3. **Доступно**: Після дозрівання переходить у доступний баланс
4. **Новий вік**: Недавно застейкані монети скидаються до 0 віку, повинні дозрівати 48 годин знову

Перевірте вкладку **Transactions**, щоб побачити винагороди за стейкінг, позначені як "Mined" або "Generated".

## Кращі практики стейкінгу

### Підтримуйте гаманець працюючим

**Стейкінг вимагає, щоб ваш гаманець працював**:
- Гаманець повинен бути відкритий і синхронізований
- Гаманець повинен залишатися розблокованим
- Комп'ютер повинен залишатися ввімкненим (або використовуйте виділену ноду для стейкінгу)

**Поради**:
- Вимкніть заставку/режим сну
- Розгляньте запуск на виділеному обладнанні
- Використовуйте VPS або комп'ютер з постійною роботою для цілодобового стейкінгу

### Оптимальне налаштування стейкінгу

**Для кращих результатів**:

1. **Підтримуйте синхронізацію гаманця**: Переконайтеся, що блокчейн повністю синхронізований
2. **Підтримуйте з'єднання**: Потрібне стабільне інтернет-з'єднання
3. **Консолідуйте малі суми**: Об'єднайте малі монети у великі (по 10+ TLR кожна)
4. **Довгий таймаут розблокування**: Використовуйте таймаут 30+ днів, щоб уникнути частого повторного розблокування
5. **Регулярні резервні копії**: Робіть резервні копії wallet.dat до і після отримання винагород

### Управління монетами

**Управління вашими стейкінг монетами**:

- **Не витрачайте недавно застейкані монети**: Дайте їм дозріти для кращих винагород
- **Консолідуйте при необхідності**: Об'єднуйте малі UTXO у великі
- **Резервуйте деякі ліквідні монети**: Тримайте деякі монети для витрат, інші для стейкінгу
- **Моніторте дозрівання**: Перевіряйте незрілий баланс на очікуючі винагороди

### Безпека при стейкінгу

**Важливі практики безпеки**:

- **Потрібен зашифрований гаманець**: Ніколи не стейкайте з незашифрованим гаманцем
- **Сильна парольна фраза**: Використовуйте 20+ символьну парольну фразу
- **Захищена система**: Переконайтеся, що комп'ютер вільний від шкідливих програм
- **Регулярне резервне копіювання**: Робіть резервні копії wallet.dat після отримання винагород
- **Обмежте віддалений доступ**: Не виставляйте RPC у публічний інтернет
- **Моніторте активність**: Регулярно перевіряйте транзакції на аномалії

## Усунення неполадок стейкінгу

### Гаманець не стейкається

**Проблема**: Монети не стейкаються, немає винагород

**Перевірте**:

1. **Гаманець зашифрований?**
   ```
   getwalletinfo
   ```
   Шукайте `"walletversion"` - повинен бути зашифрований

2. **Гаманець розблокований?**
   ```
   getwalletinfo
   ```
   Перевірте `"unlocked_until"` - повинна бути майбутня часова мітка, а не 0

3. **Монети відповідають вимогам?**
   ```
   listunspent
   ```
   - Кожна монета >= 10 TLR?
   - Confirmations >= 288?
   - Вік >= 48 годин?

4. **Гаманець синхронізований?**
   ```
   getblockchaininfo
   ```
   Перевірте, чи дорівнює `"blocks"` `"headers"`

**Рішення**:
- Розблокуйте гаманець: `walletpassphrase "passphrase" 2592000`
- Дочекайтеся дозрівання монет (48 годин)
- Переконайтеся, що гаманець повністю синхронізований
- Перевірте, що у вас є придатні монети (>= 10 TLR кожна)

### Гаманець несподівано заблокувався

**Проблема**: Гаманець заблокувався, стейкінг зупинений

**Причина**: Закінчився таймаут розблокування

**Рішення**:
```
walletpassphrase "your-passphrase" 2592000
```

Встановіть довший таймаут, щоб уникнути частого повторного блокування.

### Низька частота стейкінгу

**Проблема**: Рідкісний стейкінг блоків

**Причини**:
- Малі суми монет (< 100 TLR)
- Молодий вік монет (< 7 днів)
- Висока мережева складність

**Рішення**:
- **Накопичте більше монет**: Більше монет = вища ймовірність
- **Почекайте віку**: Монети набирають вагу протягом 90 днів
- **Консолідуйте входи**: Об'єднайте малі суми у великі
- **Будьте терплячі**: Стейкінг ймовірнісний, може зайняти час

### Незрілий баланс не підтверджується

**Проблема**: Незрілий баланс застряг

**Причина**: PoS винагороди потребують 288 підтверджень (~48 годин)

**Рішення**: Почекайте підтверджень. Перевірте:
```
getblockchaininfo
```

Якщо блокчейн синхронізований і рухається, незрілий баланс зрештою дозріє.

## Просунутий стейкінг

### Стейкінг через командний рядок

Для просунутих користувачів, що запускають `talerd` (демон без GUI):

**Запустити talerd з увімкненим стейкінгом**:
```bash
talerd -stakegen=1
```

**Розблокувати гаманець**:
```bash
taler-cli walletpassphrase "your-passphrase" 2592000
```

**Моніторинг стейкінгу**:
```bash
taler-cli getwalletinfo
taler-cli listunspent
```

### Файл конфігурації

Створіть `taler.conf` у вашому каталозі даних:

**Windows**: `%APPDATA%\TALER\taler.conf`
**macOS**: `~/Taler/taler.conf`
**Linux**: `~/.taler/taler.conf`

**Приклад конфігурації для стейкінгу**:
```ini
# Увімкнути PoS стейкінг (за замовчуванням: 1)
stakegen=1

# RPC для віддаленого управління (опціонально)
server=1
rpcuser=stakinguser
rpcpassword=your_secure_rpc_password
rpcallowip=127.0.0.1

# Мережа
listen=1
maxconnections=128
```

### Виділена нода для стейкінгу

Для серйозних стейкерів, запустіть виділену ноду для стейкінгу:

1. **Використовуйте виділене обладнання**: VPS, Raspberry Pi, або постійно працюючий ПК
2. **Встановіть TALER**: Слідуйте посібнику [Node Setup](/en/docs/node-setup/)
3. **Синхронізуйте блокчейн**: Дочекайтеся повної синхронізації
4. **Переведіть стейкінг монети**: Відправте монети у гаманець ноди
5. **Розблокуйте гаманець**: Використовуйте довгий таймаут (місяці/роки)
6. **Віддалений моніторинг**: Використовуйте RPC для перевірки статусу

**Переваги**:
- Цілодобовий стейкінг без необхідності тримати особистий комп'ютер ввімкненим
- Вищий час роботи = більше можливостей стейкінгу
- Знижені витрати на електроенергію (порівняно з майнінгом)
- Надійніший стейкінг

## Розуміння PoS vs PoW

### Порівняння

| Характеристика | Proof-of-Stake (PoS) | Proof-of-Work (PoW) |
|---------|----------------------|---------------------|
| **Обладнання** | Звичайний комп'ютер | Спеціалізоване обладнання для майнінгу |
| **Енергія** | Дуже низька | Дуже висока |
| **Бар'єр входу** | Мінімум 10 TLR | Дорогі ASIC майнери |
| **Винагороди** | 2.1 TLR за блок | 4.9 TLR за блок |
| **Час блоку** | ~140 секунд | ~60 секунд |
| **Рекомендується для** | Більшості користувачів | Професійних майнерів |

### Чому рекомендується PoS

**Для індивідуальних користувачів**:
- Не потрібне дороге обладнання
- Отримуйте пасивні винагороди на холдинги
- Допомагає захищати мережу
- Низьке споживання енергії
- Можна запустити на особистому комп'ютері

**Екологічні переваги**:
- Мінімальне використання електроенергії
- Немає електронних відходів від застарілих ASIC
- Стійкий у довгостроковій перспективі

## Економіка стейкінгу

### Мережевий розподіл

Винагороди за блоки TALER поділені:
- **70% PoW майнерам**: 4.9 TLR за блок
- **30% PoS стейкерам**: 2.1 TLR за блок

### Довгострокова стратегія

**Стейкінг як інвестиція**:
1. **Придбайте TALER**: Купіть на біржах або заробіть через роботу
2. **Тримайте в гаманці**: Переведіть у особистий гаманець
3. **Увімкніть стейкінг**: Зашифруйте і розблокуйте гаманець
4. **Реінвестуйте винагороди**: Стейкайте зароблені винагороди для складання
5. **Довгостроковий ріст**: Користуйтеся перевагами віку монет і складання

**Ефект складання**:
- Винагороди за стейкінг автоматично стають стейкаємими через 48 годин
- Більший баланс = частіший стейкінг
- Винагороди додаються до балансу, збільшуючи майбутні винагороди
- Складання збільшує холдинги з часом

## Часті запитання

### Чи блокує стейкінг мої монети?

**Ні**. Монети не заблоковані під час стейкінгу. Ви можете витратити їх будь-коли, але:
- Витрата скидає вік монети до 0
- Ви втрачаєте накопичену вагу стейкінгу
- Монетам потрібні ще 48 годин для повторного стейкінгу

### Чи можу я стейкати з декількох гаманців?

**Так**. Кожен гаманець стейкає незалежно. Ви можете:
- Запускати кілька екземплярів гаманця на різних комп'ютерах
- Розділити монети між гаманцями для надлишковості
- Кожен гаманець конкурує за блоки незалежно

### Чи потрібно тримати гаманець відкритим 24/7?

**Для кращих результатів, так**. Стейкінг вимагає:
- Працюючий і синхронізований гаманець
- Розблокований гаманець
- Активне інтернет-з'єднання

Більше часу роботи = більше можливостей стейкінгу.

### Що станеться, якщо я пропущу можливість стейкінгу?

**Нічого поганого**. Стейкінг ймовірнісний:
- У вас буде ще один шанс у наступному блоці
- Пропущені можливості не штрафують вас
- Вік монети продовжує накопичуватися
- Просто тримайте гаманець працюючим і розблокованим

### Чи можу я втратити гроші при стейкінгу?

**Немає ризику втрати монет** через сам стейкінг. Однак:
- Ризики комп'ютерної безпеки (шкідливе ПЗ, крадіжка)
- Забута парольна фраза = втрата доступу
- Немає гарантій винагород (ймовірнісно)
