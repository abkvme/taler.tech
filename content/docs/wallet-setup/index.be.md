Гэта кіраўніцтва правядзе вас праз працэс усталявання і наладкі кашалька TALER у вашай сістэме.

## Усталяванне

### Усталяванне ў Windows

1. **Распакуйце архіў**:
   - Пстрыкніце правай кнопкай мышы на загружаны файл `taler-*-win64.zip`
   - Выберыце "Выняць усё..."
   - Выберыце папку прызначэння (напрыклад, `C:\Program Files\TALER`)

2. **Знайдзіце выканальны файл**:
   - Перайдзіце ў распакаваную папку
   - Знайдзіце `taler-qt.exe` (дадатак GUI-кашалька)

3. **Апцыянальна - Стварыце ярлык на працоўным стале**:
   - Пстрыкніце правай кнопкай мышы на `taler-qt.exe`
   - Выберыце "Адправіць" → "Працоўны стол (стварыць ярлык)"

### Усталяванне ў macOS

1. **Распакуйце архіў**:
   ```bash
   tar -xzvf taler-*-macos-arm64.tar.gz
   ```

2. **Перамясціце ў Applications** (апцыянальна):
   ```bash
   sudo mv taler-*/bin/* /usr/local/bin/
   ```

3. **Прадастаўце правы на выкананне**:
   ```bash
   chmod +x /usr/local/bin/taler-qt
   ```

### Усталяванне ў Linux

1. **Распакуйце архіў**:
   ```bash
   tar -xzvf taler-*-linux-x64.tar.gz
   cd taler-*
   ```

2. **Усталюйце ў сістэму** (апцыянальна):
   ```bash
   sudo install -m 0755 -o root -g root -t /usr/local/bin bin/*
   ```

3. **Або запусціце з распакаванай дырэкторыі**:
   ```bash
   ./bin/taler-qt
   ```

## Запуск кашалька TALER

### Запуск праз камандны радок

GUI-кашалёк TALER можна запусціць з каманднага радка з рознымі опцыямі.

#### Базавы запуск

**Windows**:
```cmd
taler-qt.exe
```

**macOS / Linux**:
```bash
taler-qt
```

#### Запуск з карыстальніцкай дырэкторыяй даных

Укажыце карыстальніцкае размяшчэнне для даных блакчэйна:

```bash
taler-qt -datadir=/path/to/custom/directory
```

#### Запуск у тэставай сетцы

Для мэтаў тэставання запусціце кашалёк у тэставай сетцы:

```bash
taler-qt -testnet
```

#### Агульныя параметры каманднага радка

| Опцыя | Апісанне |
|--------|-------------|
| `-datadir=<dir>` | Указаць размяшчэнне дырэкторыі даных |
| `-conf=<file>` | Указаць файл канфігурацыі |
| `-testnet` | Выкарыстоўваць тэставую сетку |
| `-prune=<n>` | Уключыць абрэзку блакчэйна (паменшыць выкарыстанне дыска) |
| `-dbcache=<n>` | Усталяваць памер кэша базы даных у МБ (па змаўчанні: 450) |
| `-maxconnections=<n>` | Максімальная колькасць злучэнняў з вузламі (па змаўчанні: 128) |
| `-listen` | Прымаць уваходныя злучэнні (па змаўчанні: 1) |
| `-server` | Уключыць RPC-сервер |

### Запуск праз працоўны стол (GUI)

#### Windows
- Двойчы пстрыкніце на файле `taler-qt.exe` або выкарыстоўвайце ярлык на працоўным стале

#### macOS
- Перайдзіце ў распакаваную дырэкторыю
- Двойчы пстрыкніце на `taler-qt` (калі ўсталявана ў `/usr/local/bin`, запусціце з тэрмінала: `taler-qt`)
- Або запусціце з распакаванай дырэкторыі: `./bin/taler-qt`

#### Linux
- Выкарыстоўвайце сродак запуску дадаткаў для пошуку "TALER"
- Або запусціце з тэрмінала: `taler-qt`
- Або з распакаванай дырэкторыі: `./bin/taler-qt`

## Першапачатковая наладка

### Пачатковая сінхранізацыя блакчэйна

Пры першым запуску кашалька TALER ён пачне сінхранізацыю з блакчэйнам:

1. **Экран застаўкі**: Дадатак адлюструе лагатып TALER падчас загрузкі
2. **Паведамленне аб сінхранізацыі**: З'явіцца мадальнае акно з прагрэсам сінхранізацыі
3. **Індыкатар прагрэсу**: Паказвае працэнт загружанага блакчэйна
4. **Ацэнка часу**: Адлюстроўвае астатні прадбачаны час

**Заўвага**: Пачатковая сінхранізацыя можа заняць некалькі гадзін у залежнасці ад хуткасці вашага інтэрнэт-злучэння. Кашалёк павінен загрузіць усю гісторыю блакчэйна.

### Разуменне працэсу сінхранізацыі

- **Вышыня блока**: Бягучы блок супраць апошняга сеткавага блока
- **Хуткасць загрузкі**: Прапускная здольнасць сеткі ў КБ/с або МБ/с
- **Спачатку загалоўкі**: Загружае загалоўкі блокаў перад поўнымі блокамі для больш хуткай пачатковай сінхранізацыі
- **Фонавая сінхранізацыя**: Вы можаце згарнуць кашалёк, пакуль сінхранізацыя працягваецца

**Парада**: Для больш хуткай сінхранізацыі пераканайцеся, што ваша сістэма дазваляе TALER праз брандмаўэр і мае дастатковую колькасць злучэнняў з вузламі (праверце радок стану).

## Агляд інтэрфейсу кашалька

Пасля сінхранізацыі інтэрфейс кашалька TALER складаецца з некалькіх асноўных раздзелаў:

### Верхнія ўкладкі навігацыі

1. **Агляд**: Панэль кіравання, якая паказвае балансы і апошнія транзакцыі
2. **Адправіць**: Інтэрфейс для адпраўкі манет TALER
3. **Атрымаць**: Генерацыя адрасоў для атрымання манет
4. **Транзакцыі**: Поўная гісторыя транзакцый
5. **Майнінг**: Уключэнне/наладка PoW майнінгу
6. **Стэйкінг**: Наладка PoS стэйкінгу (даступна пасля сінхранізацыі)

### Радок стану (унізе)

- **Статус злучэння**: Колькасць актыўных злучэнняў з вузламі
- **Колькасць блокаў**: Бягучая вышыня сінхранізаванага блока
- **Статус сеткі**: Хэшрэйт сеткі і складанасць
- **Статус кашалька**: Статус блакіроўкі/разблакіроўкі (зашыфраваныя кашалькі)

### Галоўная панэль інструментаў

- **Файл**: Рэзервовае капіяванне кашалька, экспарт ключоў, выхад
- **Налады**: Наладка параметраў кашалька, сеткавых параметраў
- **Інструменты**: Кансоль адладкі, подпіс/праверка паведамленняў, інфармацыя аб вузлах
- **Даведка**: Дыялог "Аб праграме", параметры каманднага радка

## Стварэнне вашага першага кашалька

Дадатак TALER стварае кашалёк аўтаматычна пры першым запуску. Файл кашалька захоўваецца ў вашай дырэкторыі даных:

### Размяшчэнне кашалька па змаўчанні

- **Windows**: `%APPDATA%\TALER\wallet.dat`
- **macOS**: `~/Taler/wallet.dat`
- **Linux**: `~/.taler/wallet.dat`
